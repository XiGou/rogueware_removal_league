# Rogueware Removal League - Coding Notes

## Scope

This repository is the static multi-project site for Rogueware Removal League, a community competition for removing unwanted or rogue software. The homepage is a project directory; each project gets its own paginated ranking page.

## Architecture

- Project registry: `data/projects.json`
- Project submissions: `data/submissions/<slug>.json`
- Generator: `generate_site.py`
- Static output: `public/`
- Styles: `assets/style.css`
- Deployment: `.github/workflows/pages.yml`

The generator uses only the Python standard library. Avoid adding a package manager unless the project genuinely needs one.

## Data Contract

Each project in `data/projects.json` requires:

- `slug`
- `name`
- `title`
- `category`
- `summary`
- `description`
- `metric_label`
- `metric_unit`
- `sort`
- `rules`
- `evidence_tips`
- `safety_notice`

Each submission requires:

- `name`
- `time_seconds`
- `message`
- `evidence_url`
- `steps`
- `date`

Optional submission fields:

- `issue_url`
- `pr_url`

Adding a new project requires both a `data/projects.json` entry and `data/submissions/<slug>.json` with `[]`.

## Design Direction

Use a dense league-table style: fast to scan, high contrast, restrained motion, strong numeric hierarchy. The references are Scale Labs leaderboard density, Arena-style ranking clarity, and OpenCode's developer-tool directness.

Keep the homepage as an operational directory of projects. Keep project pages as table-first leaderboards on desktop and stacked records on mobile.

## Testing

Before shipping:

- Run `python3 generate_site.py`.
- Run `python3 -m py_compile generate_site.py`.
- Validate JSON with `python3 -m json.tool`.
- Inspect `public/index.html` and `public/<slug>/index.html` at desktop and mobile widths.
- Test pagination with more than 50 entries when changing pagination logic.
- Make sure generated HTML never injects raw user-submitted text.
