# Reference Analysis

## Scale Labs Leaderboard

Useful traits:

- Dense dark interface with strong contrast.
- Leaderboard content is visible immediately.
- Rank and metric values carry most of the visual weight.
- Cards and rows have restrained borders instead of heavy decoration.

Adaptation:

- Keep the dark league-table feel.
- Use compact project cards on the homepage and a table as the core object on each project page.
- Use color sparingly for rank, time, and proof affordances.

## Arena Leaderboard

Useful traits:

- Rankings are comparable row-by-row.
- The rank marker is always easy to locate.
- Numeric values are aligned and visually distinct from labels.

Adaptation:

- Use a compact row model on desktop.
- Keep a clear rank badge and monospace time.
- Preserve comparability after mobile stacking.

## OpenCode

Useful traits:

- Developer-facing copy is direct.
- The visual system is lightweight and fast.
- Code snippets and commands feel native to the page instead of decorative.

Adaptation:

- Keep the generator dependency-free.
- Put project and submission JSON examples close to the workflows they support.
- Avoid marketing language; explain the protocol plainly.
